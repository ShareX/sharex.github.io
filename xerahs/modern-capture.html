Modern Screen Capture

XerahS leverages the latest operating system APIs to provide high-performance, low-latency screen recording.

Windows.Graphics.Capture API

By default, XerahS utilizes the Windows.Graphics.Capture API (available in Windows 10 version 1803 and later). This modern API allows for:

  • Hardware Acceleration: Direct GPU access for capturing screen contents with minimal CPU overhead.
  • Window Exclusion: The ability to exclude the recorder's own windows or overlays from the final capture.
  • Cursor Capture: High-fidelity hardware cursor compositing.
  • High Frame Rates: Smooth 60 FPS+ recording capabilities for gaming and high-motion content.

Robust Fallback: FFmpeg

We understand that not every environment supports the latest APIs. For maximum compatibility, XerahS includes a robust fallback mechanism powered by FFmpeg.

If the modern capture APIs are unavailable (e.g., on older Windows versions) or if a specific recording configuration requires it, XerahS automatically switches to using FFmpeg directly. This ensures that you can always capture your screen, regardless of your hardware or OS version.

Did you know? This dual-engine approach allows XerahS to be lightweight and modern while maintaining the battle-tested reliability of ShareX's traditional capture methods.
